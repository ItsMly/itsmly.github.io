Zigbee2MQTT-Migrationsprobleme lösen: Network Commissioning Timed Out

Bei der Migration oder Wiederherstellung eines Zigbee-Netzwerks in Zigbee2MQTT kann folgender Fehler auftreten:

Error: Network commissioning timed out - most likely network with the same panId or extendedPanId already exists nearby

Dieses Problem entsteht häufig durch Konflikte in den Zigbee-Netzwerkeinstellungen, insbesondere bei der PAN ID, Extended PAN ID oder dem Network Key. Es kann auch vorkommen, wenn Geräte aus einem vorherigen Netzwerk noch aktiv sind. Im Folgenden wird Schritt für Schritt erklärt, wie dieses Problem behoben werden kann.

Häufige Ursachen #

  1. Zigbee2MQTT vor der Migration der Konfiguration starten:
    Wenn der Koordinator ein neues Netzwerk initialisiert, kann es zu Konflikten mit einem bereits existierenden Netzwerk kommen.

  2. Änderungen und Zurücksetzen von Netzwerk-IDs (PAN ID, Extended PAN ID oder Network Key):
    Inkonsistenzen zwischen dem Koordinator und den verbundenen Geräten können den Fehler auslösen.

  3. Aktive Zigbee-Router aus dem vorherigen Netzwerk:
    Diese Geräte versuchen möglicherweise weiterhin, sich mit der alten Netzwerkkonfiguration zu verbinden, was zu Konflikten führt.

Schritte zur Problemlösung #

1. Alle Zigbee-Router ausschalten #

Vor dem Start von Zigbee2MQTT mit einem wiederhergestellten Backup:

  • Alle Zigbee-Router (z. B. smarte Steckdosen oder Repeater) vom Stromnetz trennen.
  • Dies verhindert Störungen während des Kommissionierungsprozesses.

2. Backup wiederherstellen #

Folgende Dateien aus dem Backup wiederherstellen:

  • coordinator_backup.json: Enthält die Netzwerkkonfiguration des Koordinators.
  • database.db: Speichert Informationen und Zustände der gepairten Geräte.
  • state.json: Hält Laufzeitdaten bereit.

Diese Dateien in das Datenverzeichnis von Zigbee2MQTT kopieren (z. B. /data/zigbee2mqtt).

3. Zigbee2MQTT starten #

  • Zigbee2MQTT nach der Wiederherstellung des Backups starten und die Logs auf Fehler prüfen.
  • Sobald der Start erfolgreich war, alle Zigbee-Router wieder mit Strom versorgen.

Sollten keine Geräte nach dem Start erscheinen, Zigbee2MQTT stoppen, die Backup-Dateien erneut anwenden und den Dienst neu starten.

4. Zeit zur Stabilisierung des Netzwerks geben #

Zigbee-Netzwerke benötigen möglicherweise etwas Zeit zur Stabilisierung:

  • Etwa 10 Minuten warten, bis sich Router und Endgeräte automatisch wieder verbinden.

Wenn alle Hoffnung verloren ist – oder noch keine Geräte verbunden sind, kann ein neues Netzwerk erstellt werden, indem Folgendes zu einer Konfiguration hinzugefügt wird. Alte Einträge für pan_id, ext_pan_id und network_key müssen entfernt werden.

advanced:  
  pan_id: GENERATE  
  ext_pan_id: GENERATE  
  network_key: GENERATE
